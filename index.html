What It Prevents

Long sessions can blur the difference between what was inspected, what was said, what was approved, and what the model only assumed.

Chat becomes decision A message starts acting like project truth.
Draft becomes approved Generated material starts looking final.
Assumption becomes fact A model guess starts acting like evidence.
Instructions become content Behavior rules get mixed with project material.

How It Works

The package gives ChatGPT a frame for separating material before it becomes reusable.

It does not produce content by itself. It asks ChatGPT to inspect the bundle, track the source of what it is using, and slow down when something is about to become saved, reused, or treated as canon.

Boundary

  1. Inspect
  2. Separate
  3. Slow down
  4. Approve

Use It In A Project

A Project is recommended because the instruction anchor stays with the workspace.

  1. Zip the canon-boundary-guard-gpt folder.
  2. Add the zip to your ChatGPT Project files or sources.
  3. Copy PROJECT_CUSTOM_INSTRUCTIONS.txt.
  4. Paste it into the Project instructions.
  5. Start a new chat inside the Project.

Normal Chat

Upload the same zip and ask ChatGPT to inspect it as the active frame.

This is weaker because the frame lives only inside the conversation.

Limits

Canon Boundary Guard reduces silent promotion. It does not create hard enforcement.

  • It is not a native ChatGPT skill.
  • It is not a plugin or browser extension.
  • It cannot block every answer, save, file action, or UI action.
  • You still need to approve anything reusable, saved, or permanent.
